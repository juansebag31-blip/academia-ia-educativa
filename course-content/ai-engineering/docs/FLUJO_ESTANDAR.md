# Flujo estándar ChatGPT → NotebookLM → Codex

## Fase A — Diseño del conocimiento en ChatGPT
- Investigar y seleccionar fuentes.
- Crear el HTML fundacional.
- Revisar y aprobar contenido.
- Crear infografía y guion.
- Crear casos, evaluación y presentación.

## Fase B — Producción audiovisual en NotebookLM
- Cargar HTML, infografía, guion y fuentes.
- Generar explicación profunda.
- Revisar que cubra todos los bloques.
- Descargar el audio y convertir a MP3 cuando sea necesario.

## Fase C — Integración en Codex
- Copiar el paquete aprobado al repositorio.
- Leer AGENTS.md y el manifiesto.
- Inspeccionar el proyecto existente.
- Proponer plan.
- Implementar un módulo por vez.
- Ejecutar pruebas y revisar visualmente.

## Fase D — Validación humana
- Revisar contenido, navegación, audio, responsive y accesibilidad.
- Registrar correcciones.
- Congelar la versión aprobada del módulo.

## Fase E — Skill reutilizable
Solo después de aprobar el Módulo 1:
- detectar patrones repetibles;
- parametrizar número, título, assets y secciones;
- crear generador o skill;
- probarla con el Módulo 2;
- corregir antes de escalar a los demás.

## Responsabilidad principal de cada herramienta
- ChatGPT: arquitectura pedagógica, investigación, redacción, fuentes y diseño conceptual.
- NotebookLM: producción audiovisual fundamentada en las fuentes.
- Codex: arquitectura de software, integración, componentes, automatización, pruebas y despliegue.
