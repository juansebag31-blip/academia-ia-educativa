Selección de modelos para un copiloto de desarrollo

Este caso analiza un sistema que comprende repositorios, planifica cambios, genera código, ejecuta pruebas y revisa resultados.

1. Asignación

TareaSelecciónJustificación
Clasificar issueModelo rápidoSalida simple.
Plan arquitectónicoModelo avanzadoContexto y razonamiento.
Búsqueda semánticaEmbeddingsLocaliza código y documentación.
Generar códigoModelo de códigoSigue estructura del repositorio.
VerificarTests, linters y humanoLa calidad no depende solo del modelo.

2. Context engineering

No se envía todo el repositorio. El sistema recupera archivos relevantes, convenciones y estado del proyecto.

3. Routing

Tarea simple→Modelo rápido→Si falla→Modelo avanzado→Pruebas

4. Métricas

  • Cambios que pasan pruebas.
  • Retrabajo.
  • Coste por pull request aceptado.
  • Latencia hasta un plan útil.
  • Errores de contexto.
DecisiónLa combinación de modelos solo aporta valor si routing y verificación mejoran calidad y coste total.
