# Course Module Builder — borrador

> No utilizar como skill definitiva hasta que el Módulo 1 esté aprobado dentro de la aplicación.

## Entradas requeridas
- número y slug del módulo;
- título y objetivos;
- HTML fundacional aprobado;
- infografía PNG;
- audio MP3/M4A;
- guion o transcripción;
- casos reales;
- presentación;
- actividad, evaluación y fuentes;
- manifiesto JSON.

## Procedimiento
1. Validar que existan todos los archivos declarados.
2. Copiar assets a rutas estables.
3. Crear datos del módulo a partir del manifiesto.
4. Reutilizar los componentes aprobados del Módulo 1.
5. No reescribir contenido.
6. Añadir navegación, progreso y descargas.
7. Ejecutar lint, pruebas y build.
8. Presentar resumen de cambios y elementos faltantes.

## Criterios de calidad
- Responsive.
- Accesible.
- Audio funcional.
- Infografía ampliable.
- Fuentes visibles.
- Casos y presentación accesibles.
- Sin errores de build.
